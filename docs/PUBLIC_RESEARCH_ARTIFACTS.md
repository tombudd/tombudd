# Public Research Artifacts

This page organizes my public GitHub work as a clean-room evidence layer.

The purpose is not to publish a private system, product roadmap, or full research archive. The purpose is to show public proof-of-method: small artifacts that can be inspected, run, tested, and checked for limitations.

These repositories are useful for job search, collaborator outreach, advisor diligence, grant or investor review, research writing, and public talks.

## Governance Evaluation

**[AI Governance Benchmarks](https://github.com/tombudd/ai-governance-benchmarks)** shows reproducible evaluation design: synthetic inputs, local runners, deterministic scoring, expected reports, tests, CI, and limitation language.

Use it when the question is: can this work be measured and independently rerun?

## Agent Auditability

**[Agent Action Audit Template](https://github.com/tombudd/agent-action-audit-template)** shows operational reviewability: synthetic safe and blocked action receipts, human-review metadata, JSON schemas, negative tests, pytest validation, and CI.

Use it when the question is: can AI-assisted actions be represented as reviewable records?

## Alignment / Human Flourishing

**[Eudaimonic Alignment](https://github.com/tombudd/eudaimonic-alignment)** shows the public research thesis: AI governance should make claims about flourishing, accountability, and bounded system behavior easier to inspect.

Use it when the question is: what is the broader research direction behind the implementation work?

## Active Inference / Mathematical Scaffolding

**[Active Inference Primer](https://github.com/tombudd/active-inference-primer)** is a supporting educational artifact with minimal free-energy utilities, synthetic numerical tests, and explicit limitations.

Use it when the question is: can the underlying mathematical ideas be approached carefully and without overclaiming?

## Public Boundary

These artifacts are public-safe by design. They use synthetic examples, toy/spec material, tests, and explicit limitations.

They are proof-of-method, not disclosure-of-system. They do not claim to prove safety, alignment, deployment readiness, live autonomy, or private architecture.

The public value is simple:

- I can scope a problem.
- I can build a small artifact.
- I can test it.
- I can document limitations.
- I can protect private work while still making public claims inspectable.
